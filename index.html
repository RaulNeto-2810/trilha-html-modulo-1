Historia da WEB

Voltar ao Inicio

Primeros Computadores

A história da Web remonta aos primórdios da computação. Charles Babbage, um matemático britânico do século XIX, projetou a primeira máquina de computação mecânica, chamada de "Máquina Analítica". Embora nunca tenha sido concluída durante sua vida, os princípios que Babbage estabeleceu influenciaram profundamente o desenvolvimento posterior da computação.

Origem do Termo BUG

Em 1947, Grace Hopper, uma cientista da computação da Marinha dos Estados Unidos, encontrou um inseto preso em um dos relés da máquina Mark II. Ela removeu o inseto e o colou em seu diário, escrevendo "primeiro bug encontrado". O termo "bug" já era usado para descrever problemas técnicos, mas este foi o primeiro caso documentado de um bug literal em um sistema de computador.

Primeira Mulher Programadora

Ada Lovelace, uma matemática do século XIX, é considerada por muitos como a primeira programadora da história. Lovelace trabalhou com Charles Babbage na Máquina Analítica e escreveu o primeiro algoritmo destinado a ser processado por uma máquina, tornando-se uma pioneira no campo da computação.

Client x Server

Voltar ao Inicio

Client

  • Um cliente é um dispositivo ou programa que solicita serviços, recursos ou informações de um servidor.
  • Geralmente, em uma arquitetura cliente-servidor, o cliente é o ponto de entrada para um usuário interagir com um serviço ou aplicativo.
  • Exemplos comuns de clientes incluem navegadores da web, aplicativos de email, clientes de jogos online, etc.
  • Os clientes enviam solicitações ao servidor e esperam por respostas para realizar ações ou acessar recursos.

Server

  • Um servidor é um dispositivo ou programa que fornece serviços, recursos ou informações a clientes.
  • Ele está constantemente ouvindo por solicitações de clientes e respondendo a elas.
  • Os servidores podem fornecer uma ampla variedade de serviços, como hospedagem de sites, armazenamento de arquivos, processamento de dados, etc.
  • Os servidores executam tarefas específicas e fornecem dados ou serviços aos clientes que solicitam.

Resumo

Em resumo, enquanto o cliente solicita serviços ou recursos, o servidor fornece esses serviços ou recursos em resposta às solicitações dos clientes. Eles desempenham papéis complementares em uma arquitetura de rede, onde o cliente é o ponto de entrada para interação humana e o servidor é o fornecedor de serviços ou recursos necessários para essa interação.

Estrutura Básica do HTML

Voltar ao Inicio

O HTML (HyperText Markup Language) é a linguagem de marcação padrão usada para criar páginas da web. A estrutura básica de um documento HTML inclui as seguintes partes:

  • DOCTYPE: Declaração que define o tipo de documento e a versão do HTML.
  • HTML: Elemento raiz que contém todo o conteúdo da página.
  • Head: Elemento que contém informações sobre o documento, como título, metadados e links para arquivos externos.
  • Body: Elemento que contém o conteúdo visível da página, como texto, imagens e links.

Tags HTML

Voltar ao Inicio

As tags HTML são usadas para marcar o conteúdo de uma página da web e definir sua estrutura. Existem muitas tags HTML diferentes, cada uma com um propósito específico. Alguns exemplos comuns de tags HTML incluem:

  • p: Define um parágrafo de texto.
  • h1-h6: Define títulos de diferentes níveis de importância.
  • a: Define um link para outra página ou recurso.
  • img: Define uma imagem.
  • ul: Define uma lista não ordenada.
  • ol: Define uma lista ordenada.
  • li: Define um item de lista.

Atributos HTML

Voltar ao Inicio

Os atributos HTML são usados para fornecer informações adicionais sobre elementos HTML. Eles são especificados dentro das tags HTML e têm a forma de nome="valor". Alguns exemplos comuns de atributos HTML incluem:

  • href: Especifica o URL de destino de um link.
  • src: Especifica o URL de origem de uma imagem ou outro recurso.
  • alt: Especifica um texto alternativo para uma imagem.
  • class: Especifica uma classe CSS para um elemento.
  • id: Especifica um identificador exclusivo para um elemento.
  • style: Especifica estilos CSS inline para um elemento.

Textos

Voltar ao Inicio

Os textos são uma parte fundamental de qualquer página da web. Eles são usados para fornecer informações, instruções e conteúdo para os usuários. Alguns elementos HTML comumente usados para exibir textos incluem:

  • p: Define um parágrafo de texto.
  • h1-h6: Define títulos de diferentes níveis de importância.
  • span: Define uma seção de texto em linha.
  • strong: Define um texto em negrito.
  • i: Define um texto em itálico.
  • blockquote: Define uma citação longa.
  • del: Define um texto deletado. (Exemplo:Texto excluido com sucesso)
  • abbr: Define uma Abreviação. (Exemplo:Abreviação ou acrônimo)

Listas

Listas Ordenadas

As listas ordenadas são usadas para exibir itens em uma sequência específica. Cada item é numerado ou marcado com um número ou caractere. Alguns elementos HTML comumente usados para criar listas ordenadas incluem:

  1. Item 1
  2. Item 2
  3. Item 3

Listas Não Ordenadas

As listas não ordenadas são usadas para exibir itens em uma ordem não específica. Cada item é marcado com um marcador, como um ponto ou um círculo. Alguns elementos HTML comumente usados para criar listas não ordenadas incluem:

  • Item 1
  • Item 2
  • Item 3

Links

Voltar ao Inicio

Os links são usados para conectar páginas da web e recursos entre si. Eles permitem que os usuários naveguem entre diferentes partes de um site ou acessem conteúdo externo. Alguns elementos HTML comumente usados para criar links incluem:

  • a: Define um link para outra página ou recurso.
  • href: Especifica o URL de destino de um link.
  • target: Especifica onde abrir o link (por exemplo, em uma nova janela ou guia).
    • _blank: Abre o link em uma nova janela ou guia.
    • _self: Abre o link na mesma janela ou guia.
    • _parent: Abre o link no quadro pai.
    • _top: Abre o link no quadro superior.
  • rel: Especifica o relacionamento entre a página atual e o link.
